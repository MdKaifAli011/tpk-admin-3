Coefficient of Friction Between Body and Wedge – Complete Notes, Revision, Important Questions & Downloads

When a body descends on a smooth wedge in time t and on the rough version of the same wedge in time nt (n > 1), the ratio of times encodes the coefficient of kinetic friction. Equating the path expressions for both cases gives the elegant result μ = tanθ [1 − 1/n²]. This formula directly connects the geometric property of the incline (θ) with the timing ratio (n) measured experimentally. NEET tests this as a numerical: given θ and n, find μ. The derivation uses kinematics on rough vs smooth wedge and is a single-concept, high-mark-density topic.

Moderate
The derivation is clean but the key trap is forgetting that n is the time ratio (rough to smooth), not a speed ratio. Students who confuse n with velocity ratio get μ = tanθ[1 − n²] (wrong sign in bracket). Also, the formula assumes the body starts from rest (u = 0) and travels the same path length on both wedges. If the problem changes either assumption the formula breaks. Always re-establish conditions before applying.

NEET Mastery Strategy

1

Derive the Formula from Kinematics For smooth wedge: S = ½ g sinθ · t². For rough wedge: S = ½ g(sinθ − μ cosθ)(nt)². Set equal since path S is the same, divide sides, isolate μ. This 4-step derivation must be written from memory under exam conditions.

2

Memorise the Final Expression μ = tanθ [1 − 1/n²]. Note: when n = 1 (same time, impossible for a rough incline from rest), μ = 0 — a logical consistency check. When n → ∞ (body barely moves on rough wedge), μ → tanθ = the angle-of-repose condition.

3

Spot the Common Mistake on NEET NEET often sets n = √2 or n = 2. For n = √2: μ = tanθ[1 − 1/2] = tanθ/2. For n = 2: μ = tanθ[1 − 1/4] = 3tanθ/4. Practice these specific substitutions to avoid arithmetic errors under pressure.

1) Wedge Friction Derivation

Must Derive

A body starts from rest (u = 0) and slides down a wedge of angle θ. On smooth wedge, acceleration a₁ = g sinθ. On rough wedge, acceleration a₂ = g(sinθ − μ cosθ). Both cover the same path S; smooth takes time t, rough takes time nt. Apply S = ½a·t²: ½ g sinθ · t² = ½ g(sinθ − μ cosθ)(nt)². Dividing: sinθ = n²(sinθ − μ cosθ). Rearranging: μ = tanθ[1 − 1/n²].

  • Condition: both start from rest, same path length
  • n > 1 always (friction slows the body)
  • μ = tanθ [1 − 1/n²]
  • At n = √2: μ = tanθ/2
  • At n → ∞: μ → tanθ (angle of repose condition)
Example (NEET-style)θ = 30°, n = √2. μ = tan30°[1 − 1/2] = (1/√3)(1/2) = 1/(2√3) ≈ 0.289.

How AP Physics and US curricula compare to NEET on the wedge-timing method for finding μ.

Timing Method for μ Not in AP Physics

AP Physics would solve for μ using force balance or energy methods, never via the time-ratio formula. The specific approach of comparing descent times on smooth vs. rough incline and deriving μ = tanθ[1 − 1/n²] is exclusive to the Indian competitive exam preparation syllabus at this depth.

  • Practise deriving the formula end-to-end in under 2 minutes.
  • Know limiting cases: n = 1 (μ = 0, contradictory for rough) and n → ∞ (μ = tanθ).

Frequently Asked Questions

Notes · Downloads · Revision · Important Questions
Why does the mass of the block not appear in the formula?
In the kinematics equations, the mass m cancels from both sides when equating S on smooth and rough wedge paths. This is because both mg sinθ (driving force) and μmg cosθ (friction force) are proportional to m. The acceleration is mass-independent, and the path comparison removes m entirely.
What does it mean physically when n → ∞?
As n → ∞, the time on the rough wedge → ∞, meaning the block does not slide at all. This corresponds to μ → tanθ, which is exactly the condition for the angle of repose. The body is on the verge of sliding (static equilibrium) when μ = tanθ.
Is this formula valid if the block does not start from rest?
No. The derivation uses S = ½at² which assumes u = 0. If the block has an initial velocity u, the path equation becomes S = ut + ½at², and the time ratio n no longer simplifies to the clean formula. This is an important exam trap to watch for in problem setups.
