## Route Patterns

### Hierarchical Navigation Pattern

The main application uses a 7-level hierarchical navigation:

```
Exam → Subject → Unit → Chapter → Topic → Subtopic → Definition
```

### API Route Pattern

Most entity APIs follow this pattern:

```
/self-study/api/{entity}              → List/Create
/self-study/api/{entity}/[id]         → Get/Update/Delete
/self-study/api/{entity}/[id]/details → Get/Update Details (Content, SEO)
/self-study/api/{entity}/[id]/status  → Update Status
/self-study/api/{entity}/reorder      → Reorder (for ordered entities)
```

### Admin Route Pattern

```
/self-study/admin/{entity}      → List Page
/self-study/admin/{entity}/[id] → Detail/Edit Page
```

## Notes

1. **Base Path**: ✅ `basePath: "/self-study"` is configured in `next.config.mjs`. All routes are prefixed with `/self-study`.

2. **Authentication**:

   - Admin routes require admin authentication
   - Student routes may require student authentication
   - API routes have varying authentication requirements

3. **Dynamic Routes**: All dynamic routes use Next.js App Router dynamic segments `[param]`.

4. **API Routes**: All API routes are server-rendered and handle HTTP methods (GET, POST, PUT, DELETE).

5. **Status Routes**: Entity status routes allow toggling between "active" and "inactive" states.

6. **Reorder Routes**: Some entities support reordering via dedicated reorder endpoints.
